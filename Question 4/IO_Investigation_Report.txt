Command Executed: ls -l /proc/$$/fd

Observations: 
By listing the contents of the file descriptor directory for my current shell process, I was able to observe standard I/O streams.
The output showed descriptors 0, 1, and 2, representing standard input, standard output, and standard error.
This visualizes how the Linux kernel fundamentally manages and maps I/O operations to physical terminal endpoints.

I intentionally executed a command on a missing file to generate a system error.

Reading the log file confirmed that the "No such file" message was captured, which is vital for application logging.

Command Executed: ulimit -a

Observations: 
I executed the ulimit -a command to review the resource limits configured for my current user session.
